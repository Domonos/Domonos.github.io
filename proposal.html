Development Process and Engagement

I am making this website for other people interested in modding video games.

Testing

To test this website I will run it on my web host to make sure everything works as intended and looks as intended.

Description

The website will be about modding video games for PC.
The website is meant to inform people how to mod games along with many of the different possible gameplay changes modding brings.
The intended audience is PC gamers looking to get into modding.
I will offer text that acts as descriptions and guides. I will add images that show examples of the software needed along with images of the mods themselves and what they change or add to the games. I will also add embedded videos for the graphic changing mods.
I will get my videos and images from other websites like nexus mods and youtube.
My content will only be updated if nexus mods shuts down or is discontinued.

Growth Potential and Maintenance

Yes this website could grow as the amount of games that supports modding grows.
To be honest once the code works and the website works as intended there wont be any maintenance needed.

Organization

My content will be organized so that each page covers a topic and another page will cover a different topic.
I will have nexus mods,nexus mod manager,mod pages for gameplay changes and graphical mods as some sub topics.
My website will have a menu in the corner for navigating different pages. In the middle of the page there wil be either images or videos . Under the images or videos there will be text for descriptions.
I will have a css dropdown and also try using rounded boxes and shadows. I will also try to add a pralax scrolling effect to the pages.

Web Hosting

I will use cirteria like support for audio,video and pictures.
I wont need any special technology.
I will want javascript to be supported.

Marketing

I will not be marketing this website.
I will make sure the elements I use in my css are supported my most bornwers and use flex-box to accomodate smart phones.

Security

I do not have plans for an e-commerce component.
Content will not be password protected.
No user accounts will be needed.
There will be no user forum.
I will secure this site using github.io but other than that I wont use any security measures.
